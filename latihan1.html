1. Pendahuluan

Dokumentasi ini memandu Anda dalam memverifikasi konektivitas jaringan (menggunakan protokol ICMP atau 'ping') antara sistem operasi **Kali Linux** dan **Windows**, yang sangat penting dalam lingkungan virtualisasi (seperti VirtualBox atau VMWare) dengan konfigurasi jaringan Host-Only. Verifikasi ini memastikan kedua mesin dapat berkomunikasi sebelum melakukan pengujian atau investigasi lebih lanjut.

2. Identifikasi Alamat IP

Langkah pertama adalah mengetahui alamat IP dari kedua mesin.

Kali Linux:

ip addr show # atau: hostname -I

Catat alamat IP yang sesuai dengan antarmuka Host-Only (misalnya, 192.168.56.101).

Windows (PowerShell/CMD):

ipconfig

Catat alamat IP Host-Only Anda (biasanya merupakan alamat IP gateway virtual, misalnya, 192.168.56.1).

3. Uji Konektivitas Bidireksional (Ping)

Uji konektivitas dua arah untuk memastikan tidak ada pemblokiran firewall atau masalah konfigurasi jaringan.

Windows → Kali:

Jalankan perintah berikut di Command Prompt atau PowerShell Windows:

ping 192.168.56.101 -n 4

Kali → Windows:

Jalankan perintah berikut di Terminal Kali Linux:

ping -c 4 192.168.56.1

4. Analisis ICMP dengan Wireshark (Windows)

Kita akan menangkap paket ICMP yang bergerak saat proses ping, menggunakan Wireshark di Windows untuk menganalisis detail protokol.

  1. Buka **Wireshark** di Windows.
  2. Pilih antarmuka **Host-Only** (atau nama antarmuka yang terhubung ke jaringan virtual).
  3. Tekan tombol **Start** untuk mulai menangkap.
  4. Dari Terminal **Kali Linux**, jalankan ping ke Windows:
    ping -c 5 192.168.56.1
  5. Setelah ping selesai, tekan tombol **Stop** di Wireshark.
  6. Terapkan filter tampilan berikut untuk hanya melihat lalu lintas ICMP:
    icmp

    Atau filter yang lebih spesifik:

    ip.addr == 192.168.56.101 && ip.addr == 192.168.56.1

  7. Analisis salah satu *frame*:

    • Ethernet: Perhatikan MAC Address sumber (src) dan tujuan (dst).
    • IP: Perhatikan Alamat IP sumber (src) dan tujuan (dst).
    • ICMP: Perhatikan Tipe (Type 8 untuk echo-request, Type 0 untuk reply) dan Kode (Code 0).

5. Alternatif Penangkapan Paket di Kali

Sebagai alternatif, Anda dapat menggunakan tcpdump di Kali Linux untuk menangkap paket yang sama.

sudo tcpdump -i eth0 icmp -w icmp_cap.pcap

**(Catatan: Ganti eth0 dengan nama antarmuka Host-Only Anda, jika berbeda.)**

  1. Saat tcpdump berjalan, lakukan ping dari **Windows** ke Kali.
  2. Hentikan tcpdump dengan menekan Ctrl+C.
  3. Tampilkan hasilnya:
    sudo tcpdump -r icmp_cap.pcap

*(Opsional: Anda dapat memindahkan file icmp_cap.pcap ke Windows dan membukanya di Wireshark untuk analisis visual.)*

6. Verifikasi dan Hasil

Pengujian Anda berhasil jika mencapai hasil berikut:

  • Ping bidireksional berhasil (tidak ada *timeout*).
  • Terlihat paket Echo Request (Tipe 8) dan Echo Reply (Tipe 0) di Wireshark.
  • Anda dapat menjelaskan hubungan antara MAC Address, IP Address, dan detail tipe/kode ICMP dari paket yang ditangkap.

Untuk pembersihan, hentikan semua proses penangkapan (`Wireshark`/`tcpdump`) dan simpan file icmp_cap.pcap untuk laporan jika diperlukan.

7. Pemecahan Masalah (Troubleshooting)

Jika terjadi **Ping timeout** atau kegagalan konektivitas, periksa hal-hal berikut:

  • Pastikan mode adaptor jaringan virtual diatur ke **Host-Only**.
  • Periksa **Windows Firewall** (coba nonaktifkan sementara untuk pengujian, lalu aktifkan kembali).
  • Periksa status antarmuka di Kali Linux. Jika antarmuka belum aktif, jalankan perintah ini di Terminal Kali:
    sudo ip link set eth0 up
