1. Pendahuluan & Prasyarat

Dokumentasi ini akan memandu Anda dalam melakukan penangkapan paket menggunakan Wireshark untuk menganalisis pola lalu lintas jaringan normal (HTTP) dan membandingkannya dengan pola serangan Denial of Service (DoS) sederhana, yaitu ICMP Flood.

LAB ONLY: Simulasi serangan hanya boleh dilakukan pada jaringan yang terisolasi (Host-Only/isolated network).

Topologi & Persiapan

  • Server HTTP (Kali): 192.168.56.101 (Contoh IP)
  • Klien/Sniffer (Windows): 192.168.56.1 (Contoh IP)
  • Verifikasi Python 3 di Kali:
    python3 --version
  • Pastikan Windows sudah siap untuk menjalankan Wireshark.

2. Menjalankan Server HTTP di Kali

Kita akan menggunakan modul http.server Python untuk membuat server web sederhana di Kali pada port 8080.

  1. Buat direktori kerja dan buat file index.html:
    mkdir -p ~/web_test && cd ~/web_test echo "Hello from Kali" > index.html
  2. Jalankan server HTTP. **Biarkan terminal ini tetap terbuka.**
    python3 -m http.server 8080

3. Mengakses & Menangkap HTTP (Windows)

Gunakan Wireshark di Windows untuk menangkap sesi HTTP normal.

  1. Di **Windows (Wireshark)**: Pilih antarmuka **Host-Only** → Tekan **Start** untuk mulai menangkap.
  2. Di **Windows (Browser/PowerShell)**: Akses server yang baru saja dibuat:
    powershell curl http://192.168.56.101:8080 # atau buka di browser favorit Anda: http://192.168.56.101:8080
  3. Setelah akses, terapkan filter tampilan di Wireshark:
    http # atau tcp.port == 8080

Cek: Anda seharusnya melihat paket yang menunjukkan proses TCP Three-way Handshake (SYN, SYN-ACK, ACK), diikuti oleh permintaan **GET / HTTP/1.1**, dan respons **200 OK**.

4. Simulasi ICMP Flood

Gunakan terminal Kali baru untuk mensimulasikan serangan ICMP Flood (ping berkecepatan tinggi) ke IP Windows.

Instalasi Tools (Jika Diperlukan):

sudo apt update && sudo apt install -y hping3

Simulasi Flood (Pilihan 1: hping3 - Cepat & Intens)

sudo hping3 --flood -1 192.168.56.1 # Gunakan Ctrl+C untuk menghentikan

Simulasi Flood (Pilihan 2: ping - Lebih Ringan)

ping -f -i 0.002 192.168.56.1
  1. Di Wireshark Windows, biarkan penangkapan tetap berjalan saat Anda menjalankan serangan *flood* selama beberapa detik.
  2. Hentikan penangkapan di Wireshark.
  3. Terapkan filter:
    icmp
  4. Lakukan analisis grafis melalui **Statistics → I/O Graph** dan bandingkan *packets per second* (pps) antara periode sebelum dan selama *flood*.

5. Verifikasi dan Analisis Hasil

Berdasarkan penangkapan paket, Anda harus mengamati pola lalu lintas yang berbeda:

  • Pola HTTP Normal: Ditandai dengan urutan paket yang jelas (burst pendek) yang melibatkan **TCP Handshake** diikuti oleh pasangan **Request/Response**.
  • Pola ICMP Flood: Ditandai dengan aliran paket ICMP yang **terus-menerus** dan memiliki nilai **pps yang sangat tinggi**, tanpa adanya mekanisme *handshake* yang terstruktur.

6. Pembersihan & Troubleshooting

Pembersihan

  • Tekan Ctrl+C untuk menghentikan proses **hping3** atau **ping -f** di Kali.
  • Tekan Ctrl+C untuk menghentikan server HTTP Python.
  • Simpan hasil penangkapan Wireshark (.pcap) untuk dokumentasi.

Pemecahan Masalah (Troubleshooting)

  • Perintah Ditolak: Perintah hping3 atau ping -f memerlukan hak akses root. Pastikan Anda menggunakannya dengan sudo.
  • Port 8080 Tidak Terlihat: Pastikan Anda menggunakan filter yang benar di Wireshark (http atau tcp.port == 8080) dan memastikan alamat IP serta port diakses dengan benar di Windows (http://192.168.56.101:8080).
